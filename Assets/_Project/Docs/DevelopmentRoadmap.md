# 개발 로드맵 (Development Roadmap)

**프로젝트**: MatchBattle
**목표**: MVP 완성 및 플레이 테스트
**기간**: 3주 (2025-12-12 ~ 2025-12-31)
**엔진**: Unity 6+

---

## 📋 전체 개요

### 개발 목표
- ✅ 플레이 가능한 MVP 빌드 완성
- ✅ 핵심 게임플레이 루프 구현 (보드 → 전투 → 보상 → 반복)
- ✅ 3종 적 + 10종 블록 구현
- ✅ 기본 밸런싱 완료
- 🔶 내부 플레이 테스트 진행

### 개발 우선순위
1. **핵심 게임플레이** (Week 1-2): 보드 + 전투 시스템
2. **콘텐츠 확장** (Week 2): 블록/적 다양화
3. **밸런싱 & 피드백** (Week 3): 플레이 테스트 및 개선

---

## 📅 주차별 로드맵

### Week 1: 프로토타입 (2025-12-12 ~ 2025-12-18)

#### 목표
기본 게임플레이 루프가 작동하는 프로토타입 완성

#### Day 1-2: 프로젝트 셋업 & 보드 시스템 기초
```
✅ 프로젝트 구조 설정
  - Assets/_Project/ 폴더 구조 생성
  - Core, Board, Combat, Data, UI 폴더 분리
  - ScriptableObject 기반 데이터 구조 설계

✅ 보드 시스템 - 기초 구현
  - 8×8 그리드 생성 (Grid<Block>)
  - 블록 생성 및 배치
  - 블록 2종 구현: 🔴⚔️검, 🔵🛡️방패
  - BlockData ScriptableObject 생성
```

**완료 기준**: 8×8 보드에 빨강/파랑 블록이 랜덤 생성됨

#### Day 3-4: 보드 시스템 - 입력 & 연결
```
✅ 입력 처리
  - 터치/마우스 드래그 감지
  - 그리드 좌표 변환

✅ 경로 연결 시스템
  - 인접 판정 (상하좌우)
  - 같은 색상 판정
  - 최소 3개 연결 규칙
  - 경로 시각화 (라인 렌더러)

✅ 블록 제거 & 재생성
  - 연결된 블록 제거
  - 블록 드롭 애니메이션
  - 빈 칸 채우기
```

**완료 기준**: 같은 색 3개 이상 드래그 연결 시 제거 & 재생성

#### Day 5-6: 전투 시스템 기초
```
✅ 플레이어 시스템
  - HP, Defense 스탯 관리
  - 방어력 계산 (최대 30)

✅ 적 시스템 - 슬라임만
  - Enemy ScriptableObject
  - 기본 공격 패턴
  - HP 관리

✅ 턴 관리
  - 플레이어 턴 / 적 턴 구분
  - 블록 효과 → 데미지 계산
  - 적 행동 실행

✅ 승패 판정
  - 적 HP 0 → 승리
  - 플레이어 HP 0 → 패배
```

**완료 기준**: 슬라임과 1:1 전투 가능, 승패 판정 작동

#### Day 7: Week 1 통합 테스트
```
✅ 전체 게임플레이 루프 테스트
  - 보드에서 블록 연결
  - 공격/방어 효과 발동
  - 적 턴 진행
  - 전투 종료 및 재시작

✅ 버그 수정 및 안정화
```

**Week 1 완료 기준**:
- [ ] 빨강(검) 블록으로 슬라임 공격 가능
- [ ] 파랑(방패) 블록으로 방어력 획득 가능
- [ ] 슬라임의 공격이 플레이어 HP/방어력 감소
- [ ] 승리/패배 후 재시작 가능

### Week 2: 콘텐츠 확장 (2025-12-19 ~ 2025-12-25)

#### 목표
블록 종류와 적 다양화, 전투 깊이 추가

#### Day 1-2: 블록 시스템 확장
```
✅ 5가지 색상 블록 추가
  - 🔴 붉은 블록: 검, 도끼, 화염
  - 🔵 푸른 블록: 방패, 회피, 반격
  - 🟡 노란 블록: 금화, 보석, 보너스
  - 🟤 갈색 블록: 쓰레기, 포션, 버프
  - 🟣 보라 블록: 별 (와일드카드, 선택)

✅ BlockPool 시스템 구현
  - 가중치 기반 블록 생성
  - 블록 출현 확률 조정

✅ 효과 시스템
  - 공격/방어 기본 효과
  - DOT (화염)
  - 회복 (포션)
  - 골드 획득
```

**완료 기준**: 10종 블록이 설정된 확률로 생성되며 각 효과가 정상 작동

#### Day 3-4: 전투 시스템 확장
```
✅ 적 3종 추가
  - 슬라임 (HP 30, 공격 5)
  - 고블린 (HP 50, 공격 8, 강공격)
  - 오크 보스 (HP 150, 공격 15, 분노)

✅ 적 행동 패턴
  - 행동 풀 시스템
  - 다음 행동 예고
  - 강공격 예고 UI

✅ 상태 효과
  - DOT (화상, 독)
  - 버프 (공격력 증가)
  - 디버프 (방어 감소)

✅ 특수 능력
  - 오크 분노 (HP 50% 이하)
  - 회피/반격 메카닉
```

**완료 기준**: 3종 적과 전투 가능, 각 적의 패턴과 특수능력 작동

#### Day 5-6: 블록 빌딩 시스템
```
✅ 보상 시스템
  - 전투 승리 시 보상 선택
  - 블록 추가/업그레이드
  - BlockPool 동적 변경

✅ 진행 시스템
  - 간단한 맵 (3-4전투)
  - 전투 → 보상 → 전투 루프
  - 최종 보스 (오크) 배치

🔶 유물 시스템 (선택)
  - 기본 유물 2-3개
  - HP 최대치 증가
  - 공격력 배율 증가
```

**완료 기준**: 전투 승리 → 블록 추가 선택 → 다음 전투에서 해당 블록 출현

#### Day 7: Week 2 통합 테스트
```
✅ 전체 플레이 루프 테스트
  - 첫 전투(슬라임) → 블록 추가
  - 두 번째 전투(고블린) → 블록 추가
  - 보스 전투(오크) → 승리

✅ 밸런싱 1차 조정
  - 블록 효과 수치
  - 적 HP/공격력
  - 전투 길이 측정
```

**Week 2 완료 기준**:
- [ ] 10종 블록 모두 구현 및 작동
- [ ] 3종 적과 전투 가능
- [ ] 블록 빌딩으로 BlockPool 변화 확인
- [ ] 슬라임 → 고블린 → 오크 순서로 진행 가능

## 🔗 시스템 간 의존성

### 보드 시스템 → 전투 시스템
전투 시스템이 보드의 블록 효과를 활용하여 데미지/방어력을 계산합니다.

**구현 순서**: 보드 먼저 → 전투 나중

### 전투 시스템 → 블록 빌딩 시스템
블록 빌딩 시스템이 전투 보상으로 BlockPool을 동적으로 변경합니다.

**구현 순서**: 전투 먼저 → 블록 빌딩 나중

### 모든 시스템 → UI 시스템
UI는 모든 게임 시스템의 상태를 시각적으로 표시하는 프레젠테이션 레이어입니다.

**구현 순서**: 핵심 시스템 먼저 → UI 나중

## ⚠️ 리스크 관리

### 리스크 1: 밸런싱 시간 부족
**위험도**: 중
**영향**: 게임이 너무 쉽거나 어려워 재미 감소
**완화 방안**:
- Week 2부터 지속적 플레이 테스트
- 수치를 데이터(SO)로 관리해 빠른 조정 가능
- 블록/적 효과를 단순하게 유지 (복잡도 ↓)

### 리스크 2: 보드 매칭 알고리즘 복잡도
**위험도**: 중
**영향**: 성능 저하, 버그 증가
**완화 방안**:
- 8×8 그리드로 작게 시작 (64칸만 검사)
- 경로 연결은 단순 리스트 관리
- 최대 연결 수 제한 (20개)

### 리스크 3: 적 AI 복잡도
**위험도**: 낮
**영향**: 개발 시간 증가
**완화 방안**:
- MVP는 단순 패턴만 (랜덤 선택)
- 행동 풀을 데이터로 관리
- 복잡한 AI는 Week 3 이후로 연기

### 리스크 4: Unity 6 버전 이슈
**위험도**: 낮
**영향**: 예상치 못한 기술적 문제
**완화 방안**:
- 검증된 기능만 사용 (Grid, ScriptableObject)
- 실험적 기능 사용 금지
- 정기적 백업 및 버전 관리

## 🎯 성공 기준

### MVP 완료 조건
1. ✅ **플레이 가능**: 처음부터 끝까지 크래시 없이 플레이
2. ✅ **핵심 루프**: 보드 매치 → 전투 → 보상 → 반복 작동
3. ✅ **콘텐츠**: 10종 블록, 3종 적 구현
4. ✅ **밸런싱**: 평균 승률 60-80%, 플레이 시간 10-15분
5. ✅ **안정성**: 크리티컬 버그 0개

### 품질 기준
- **성능**: 60fps 유지 (Windows, 1080p)
- **UX**: 조작 후 0.1초 이내 시각적 피드백
- **밸런싱**: 3회 플레이 중 2회 승리 (숙련 플레이어 기준)

### 테스트 기준
- **단위 테스트**: 각 시스템별 기능 테스트 통과
- **통합 테스트**: 전체 게임플레이 루프 10회 이상 플레이
- **빌드 테스트**: Windows 빌드 실행 및 안정성 확인
