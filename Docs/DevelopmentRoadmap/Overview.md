# MatchBattle 개발 로드맵 - 전체 개요

**작성일**: 2025-12-18
**예상 기간**: 6주 (42일)
**목표**: 플레이 가능한 로그라이크 매치-3 RPG 완성

---

## 🎯 프로젝트 비전

**장르**: 로그라이크 매치-3 RPG
**핵심 게임플레이**: 보드 매치 → 전투 효과 → 적 턴 → 반복
**목표 경험**: 전략적 블록 매칭 + 긴장감 있는 턴제 전투

---

## 🎮 Phase별 핵심 목표

### Phase 1: MVP 완성 (2주)
**목표**: 완전한 단일 적 전투 경험

**핵심 작업**:
1. 플레이어/적 공통 로직 통합
2. 블록 효과 완전 구현 (검/도끼/화염/버프)
3. 버프/디버프 로직 완성
4. 플레이어 상태 효과 시스템

**마일스톤**: 단일 전투가 완전히 재밌음

**세부 문서**: [Phase1_MVP.md](Phase1_MVP.md)

---

### Phase 2: 시스템 확장 (2주)
**목표**: 완전한 게임 루프 (전투 → 보상 → 맵 → 전투)

**핵심 작업**:
1. 다중 적 전투 시스템 (최대 4체)
2. 보상 선택 시스템 (유물/골드/회복)
3. 맵 시스템 (노드 선택, 경로)
4. 상점 시스템

**마일스톤**: 전투 → 맵 → 전투 루프 동작

**세부 문서**: [Phase2_Expansion.md](Phase2_Expansion.md)

---

### Phase 3: Polish (2주)
**목표**: 출시 가능한 품질

**핵심 작업**:
1. 애니메이션 및 이펙트
2. 사운드 시스템 (BGM, 효과음)
3. 메타 진행 시스템 (영구 업그레이드)
4. 런 통계 및 업적

**마일스톤**: 플레이어가 만족할 수 있는 완성도

**세부 문서**: [Phase3_Polish.md](Phase3_Polish.md)

### 다음 작업 (Phase 2 계속)
1. **보상 선택 시스템** (1-2일, 중간) ⭐
   - 전투 승리 후 3가지 보상 중 선택
   - 유물/골드/회복 등

2. **유물 시스템** (2-3일, 중간)
   - ScriptableObject 기반 유물 데이터
   - 패시브 효과 적용

3. **맵 시스템** (3-4일, 높음)
   - 노드 기반 진행
   - 전투/상점/휴식/이벤트 노드

4. **상점 시스템** (1-2일, 낮음)
   - 골드로 블록/유물 구매

## 💡 성공 기준

### 기술적 성공
- [ ] 모든 핵심 기능 동작
- [ ] 치명적 버그 없음
- [ ] 60 FPS 유지
- [ ] 메모리 누수 없음

### 게임플레이 성공
- [ ] 한 판에 10-15분 소요
- [ ] 전투가 전략적이고 재밌음
- [ ] 보상 선택이 의미 있음
- [ ] 런 실패 후에도 진행감 있음

### 사용자 경험 성공
- [ ] 직관적인 UI/UX
- [ ] 빠른 피드백 (애니메이션, 사운드)
- [ ] 튜토리얼 없이도 이해 가능
- [ ] 재플레이 가치 있음

## 🚨 리스크 관리

### High Risk
- ~~**다중 적 전투**: 기술적 복잡도 높음~~ ✅ 해결됨

- **맵 시스템**: 구현 시간 오래 걸림
  - **완화**: 선형 진행으로 단순화, 점진적 확장

### Medium Risk
- **블록 효과 밸런싱**: 반복 테스트 필요
  - **완화**: 수치를 쉽게 조정 가능하게 설계

- **애니메이션**: 시간 부족 가능
  - **완화**: 기본만 구현, 나머지는 선택사항

### Low Risk
- ~~**버프/디버프**: 시스템 이미 구축됨~~ ✅ 완료
- **상점**: 단순한 구조
- **보상 시스템**: 기본 구조는 단순함
